# npm のことについて理解したいのでしらべる

- node.js package manager
- 言語のパッケージ管理ツール

- パッケージ管理ソフトを使用する
  　依存関係を解消してくれる
  　必要なソフトやライブラリ（下記依存関係）のひとまとめをパッケージといい、
  　このパッケージを管理してくれる
  　パッケージを使用すると、必要な依存関係を自力で解決してくれる
  　必要なソフト、ライブラリを適切にインストールしてくれる
- パッケージ管理システムを使う利点
  　とにかく使いやすい
  　削除する時も便利
  ※なんか難しいこと言ってるけど、要するに必要なモジュールを管理してくれる
  よく考えたら npm または yarn でインストールしてから、プロジェクト開始することが多い
  ということは npm または yarn でインストールするものはすべてパッケージ管理システムということ
  ということはパッケージ名＝パッケージ管理システム

# npm と yarn の違い

- 大義としては各種 JavaScript のパッケージマネージャー。理解はさておき、基本的な違い、各種メリット、デメリットのみザックリまとめる

- パッケージ管理ソフトを使用しない
  　自分で依存関係を解決しないといけない
  　依存関係とはこのソフトにはこのバージョンの〇〇が必要だよねというソフト同士の関係のこと
  　パッケージ管理をソフトに依存しない場合、手動で実施しないといけない
  　手動で実施するにはこれらの依存関係をしっかりと自分で管理し、必要なソフトを
  　準備したり、削除する時に気を使わないといけない
