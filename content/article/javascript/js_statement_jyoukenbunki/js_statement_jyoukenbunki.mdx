  - 文を正しく理解すれば JavaScript の基本も理解できる

# 基礎基本を抑える

## JavaScript の条件分岐[文]

- if 文
  - 条件式の評価結果が'true'であるならば'実行する文'が実行される

```js
if (true) {
  console.log("実行する文")
}
if (1) {
  console.log("jyoukensiki = 'true' ->  文は実行される ")
}
if ("") {
  console.log("jyoukensiki = 'false' ->  文は実行されない ")
}
if (0) {
  console.log("jyoukensiki = 'false' ->  文は実行されない ")
}
```

- if 文の条件式は 'true' または 'false' 以外の値の場合
- その値を暗黙的に 'true' または 'false' へ変換してから条件式として判定する
- 'true' または 'false' = 真偽値という
- 真偽値へ変換すると 'true' と判定される値の種類が多い
- 'false' と判定される値 = falsy
- falsy = [false, undefined, null, 0, 0n, NaN, ""];

- else if 文
- 複数の条件分岐を書く場合 = if + else if

```js
const version = 2
if (version === 1) {
  console.log(1)
} else if (version === 2) {
  console.log(2)
} else if (version === 3) {
  console.log(3)
}
```

- else 文
- 条件一致 = [if,else if]文
- 条件が一致しない = [else]文

```js
const number = 6
if (number < 3) {
  console.log(`number is greater than 3: ${number}`)
} else if (number < 5) {
  console.log(`number is greater than 5: ${number}`)
} else {
  console.log(`number is over 5: ${number}`)
}
```

- ネストした if 文
- statement = [if,else of,else] = ネストして書くことができる

```js
const year = new Date().getFullYear()
if (year % 4 === 0) {
  if (year % 100 === 0) {
    if (year % 400 === 0) {
      console.log(`${year}is a leap year.`)
    } else {
      console.log(`${year} is not a leap year.`)
    }
  } else {
    console.log(`${year}is a leap year.`)
  }
} else {
  console.log(`${year} is not a leap year.`)
}
```

- switch 文

  - switch 文は、if 文と同様に式の評価結果に基づく条件分岐を扱う
  - break 文は、switch 文から抜けて switch 文の次の文から実行するためのもの
  - default 文は、どの case にも該当しない場合の処理を実行するためのもの

```js
const version = "ES6"
switch (version) {
  case "ES5":
    console.log("ECMAScript 5")
    break
  case "ES6":
    console.log("ECMAScript 2015")
    break
  case "ES7":
    console.log("ECMAScript 2016")
    break
  default:
    console.log("しらないバージョンです")
    break
} // > "ECMAScript 2015"と出力される
// break;の後、switchから抜けて実行される次の文
```

# まとめ　ポジティブ意見

- if 文、else if 文、else 文で条件分岐した処理を扱える
- 条件式に指定した値は真偽値へと変換してから判定される
- 真偽値に変換すると false となる値を falsy と呼ぶ
- switch 文と case 節、default 節を組み合わせて条件分岐した処理を扱える
- case 節で break 文しない場合は引き続き case 節が実行される
- 条件分岐には if 文や switch 文を利用する
  - 複雑な条件を定義する場合には、if 文のネストが深くなりやすい
  - そのような場合には、条件式自体を見直してよりシンプルな条件にできないかを考えてみることも重要
