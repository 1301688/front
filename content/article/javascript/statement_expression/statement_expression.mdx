## JavaScript は文（statement)と式（expression）から構成されている

# 基礎基本を抑える

- 式 = 値を生成し変数に代入できるもの
- 種類 = リテラル・変数・関数式・式と演算子の組み合わせ
- 式の特徴 = 式を評価すると結果の値を得ることができる
- 評価した結果を変数に代入できるものは式であるという理解で問題ない

```js
console.log(1)
const hensu_ = "hensu_"
const fn = function () {
  return 1
}
const total = 1 + 1
```

- 文 = 処理する 1 ステップが 1 つの文
- 種類 = if 文や for 文など
- 文は式ではないため、if 文を変数に代入することはできない認識で OK

```js
const istrue = true
if (istrue) {
  console.log("seikai") // 実行される文
}
```

- 式文
- 文となった式のことを式文という
- 基本的に文が書ける場所には式を書ける
- 文は式になれますが、if 文のように文は変数に代入できないので式になれない

- ブロック文
- 文を{と}で囲んだ部分をブロックという
- ブロックには複数の文が書ける

```js
if (true) {
  console.log("文1")
  console.log("文2")
}
```

## 構成２　内向けの場合

1. 記事構成（記事構成）
2. 目的（解決したい問題）結論から宣言する
3. 解決策（対策手順の概要）
4. ユースケースのイメージ（ストーリーこれ ② かも）
5. 説明と理由（手順を段階的に）（理由を書くとイメージがさらに UP する）（関連記事へのリンク＝自分用、関連記事への後記＝稼ぎ用）
6. 実習（コード挿入）
7. まとめ　まとめるならポジティブ　まとめないケースもあるマナブさん
8. 稼ぐ要素を必ず添付
9. 応用リンク（⑤ の説明の後記）

## ひとつひとつ理解するための大義、フレームワーク

1. 目的と issue 本当に達成すべきか
2. 問題と改善 どのような問題を解決してくれるか
3. 基本の使い方 基本的な使い方だけ書く
4. 応用の使い方 応用して使うにはどのような知識を持っておけば良いかだけ書く
   ※前提として 1,2 のみで大枠だけ書くだけ
   ※応用は改善時に学習しないと定着不可能のため
